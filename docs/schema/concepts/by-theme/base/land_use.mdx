Land Use Type
====

Features in the `land_use` type within Overture's `base` theme come form OpenStreetMap. The **subtype** and **class** of an Overture feature is determined by the tags that are present in OpenStreetMap.

Below is the logic that is currently used to convert from OSM tags into Overture schema.


<details>
    <summary>How do I interpret this query?</summary>

    Each `WHEN` line in the `CASE` statement is a condition that defines both the **subtype** and the **class** of a feature. For example:
    ```sql
        WHEN element_at(tags,'landuse') IN ('residential', 'static_caravan', 'garages')
            THEN ROW('residential', element_at(tags,'landuse'))
    ```

    Here, `element_at(tags, 'landuse')` is accessing the value of the `landuse` tag in OSM. So, if landuse= `residential`, `static_caravan`, or `garages` then the subtype will be **residential** and the class will be either `residential`, `static_caravan`, or `garages`.

    Since this particular statement is at the top of the list, it will take priority over other tags on the feature.
</details>
